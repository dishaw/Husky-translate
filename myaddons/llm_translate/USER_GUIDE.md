# LLM Translate 用户使用说明

## 1. 模块用途

LLM Translate 是一个 Odoo 文档翻译模块，用于把 Word、PowerPoint、PDF 和图片文件拆分成可逐段处理的翻译任务，并通过已配置的 LLM 服务完成翻译。模块支持术语库、分段续译、人工编辑、失败重试、双栏预览、双语导出和图片 OCR 覆盖层编辑。

## 2. 技术栈概览

- 运行平台：Odoo 18 模块
- 后端语言：Python
- 后端框架：Odoo ORM、Odoo HTTP JSON Controller
- 前端框架：Odoo Web Client Action、OWL Component
- 前端资源：JavaScript、XML template、SCSS
- LLM 能力：依赖本系统的 `llm`、`llm_thread`、`llm_tool`、`llm_assistant` 模块
- 知识库/术语库：依赖 `llm_knowledge`
- 项目归档：依赖 Odoo `project`
- DOCX 处理：`python-docx`，并结合原 DOCX XML 结构做高保真重建
- PPTX 处理：`python-pptx`
- PDF 处理：PyMuPDF，Python 包名通常为 `PyMuPDF`，代码中导入为 `fitz`
- 老格式转换：LibreOffice headless，用于 `.doc`、`.ppt` 等旧格式转为现代 Office 格式
- 图片/PDF OCR 翻译：通过多模态 LLM 模型处理图像内容，并保存 OCR 文本框位置

## 3. 入口位置

在 Odoo 左侧菜单中进入：

- LLM > Translation：打开翻译工作台
- LLM > Configuration > Translations：查看翻译任务列表

任务列表中可查看任务名称、源文件、语言、供应商、模型、项目、进度、状态、文件大小和创建时间。

## 4. 创建翻译任务

1. 打开 Translation 页面。
2. 选择项目。源文件和译文会关联到该项目。
3. 选择源语言和目标语言。
4. 选择 LLM Provider 和 Model。
5. 如有术语库需求，选择 Glossary Knowledge Base。
6. 上传文件。
7. 系统会创建翻译任务并抽取文档内容。

支持的文件类型包括：

- Word：`.doc`、`.docx`
- PowerPoint：`.ppt`、`.pptx`
- PDF：`.pdf`
- 图片：`.jpg`、`.jpeg`、`.png`、`.bmp`、`.gif`、`.webp`、`.tiff`、`.tif`、`.svg`

注意：图片 OCR、PDF 页面图像识别等功能需要选择支持视觉/多模态能力的模型。

## 5. 开始翻译

打开任务后点击 Start/Translate 开始翻译。系统会按智能批次逐段翻译：

- 空行会自动跳过。
- 纯数字、单位、编号等非翻译内容会直接复制。
- 长段落会按句子拆分，避免超过模型上下文限制。
- 表格会按单元格翻译，尽量保持表格结构。
- 翻译中断后可以再次点击开始继续。

如果连续出现 5 个翻译错误，页面会弹出确认框，询问是否终止当前翻译。选择终止后，任务会暂停，已完成内容保留，后续可以继续或重试错误段落。

## 6. 预览和编辑

任务详情页提供类似 Word 的预览界面：

- Split 模式：左侧原文，右侧译文。
- Bilingual 模式：原文和译文上下对照。
- 可以调整缩放比例。
- 可以直接编辑原文或译文段落。
- 修改译文后会自动保存。
- 单段可以重新翻译。
- 失败段落可以重试。

编辑译文时，系统会尝试从人工修订中学习术语，更新术语库条目，用于后续翻译一致性。

## 7. 表格翻译

表格内容会按行和单元格提取。导出双语 DOCX 时，表格采用交错行结构：

1. 原文行
2. 对应译文行
3. 下一行原文
4. 对应译文行

这样便于逐行核对表格内容。

## 8. 图片和 OCR 翻译

对于图片或含图片的文档：

1. 系统会把图片交给多模态模型识别文字。
2. 识别出的文本会生成可编辑的译文覆盖层。
3. 用户可以拖动文本框、调整字号、修改译文、删除文本框。
4. 修改后的 OCR 文本框会保存到任务中。
5. 导出时会尽量把译文覆盖回图片或 PDF 页面。

如果 OCR 报错，优先检查当前模型是否支持图片输入。

## 9. 术语库

术语库用于保持专有名词、技术词汇和固定表达的一致翻译。

可用操作：

- 查看当前语言对的术语数量。
- 手动新增术语。
- 编辑术语译文。
- 删除术语。
- 从人工修改中自动学习术语。

翻译时，系统会把相关术语作为上下文提供给 LLM。

## 10. 导出结果

翻译完成后可以下载结果文件。

常见导出行为：

- DOCX：优先基于原文档结构替换或插入译文，尽量保留模板、样式、表格、图片和页眉页脚。
- 双语 DOCX：基于原 DOCX 克隆结构，在原文段落后插入译文段落；表格按一行原文一行译文插入。
- PPTX：基于原 PPTX 替换文本。
- PDF：基于原 PDF 页面和 OCR 坐标覆盖译文。

如果有部分段落失败，系统仍可能生成部分译文文件，任务状态会显示为 Error，可后续重试失败段落。

## 11. 常见问题

### 上传后提示格式不支持

确认文件扩展名是否在支持列表中。旧版 `.doc` / `.ppt` 需要服务器安装 LibreOffice。

### 图片或 PDF OCR 失败

确认当前模型支持多模态/视觉输入。普通纯文本模型通常不能处理图片。

### 翻译中断

重新打开任务并点击开始翻译即可继续。已经完成的段落不会重复翻译，错误段落可单独重试。

### 译文格式和原文不完全一致

DOCX 和 PPTX 会尽量复用原文档结构和样式。复杂文档中的文本框、合并单元格、编号、浮动图片等元素可能需要人工复核。

### 双语导出标题编号异常

译文段落会禁用原文自动编号，避免标题编号翻倍。如果个别复杂模板仍显示异常，建议检查 Word 标题样式中的自动编号设置。

### 下载受限

临时/访客用户可能不能下载译文，需要登录正式账号。

## 12. 推荐使用流程

1. 先选择正确的 Provider 和 Model。
2. 图片、扫描 PDF 或含图片文档优先选择多模态模型。
3. 上传文件并等待抽取完成。
4. 开始翻译。
5. 翻译完成后检查失败段落。
6. 人工修订关键术语和重点段落。
7. 重试错误段落。
8. 下载译文或双语文件。
9. 将人工修订沉淀到术语库，供下次翻译复用。
